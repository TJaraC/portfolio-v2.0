<task type="auto">
  <name>Task 2: Add missing Geist-Regular.woff2 and Geist-Medium.woff2 font files</name>

  <action>
The build warning is:
```
/fonts/Geist-Regular.woff2 referenced in /fonts/Geist-Regular.woff2 didn't resolve at build time
/fonts/Geist-Medium.woff2 referenced in /fonts/Geist-Medium.woff2 didn't resolve at build time
```

Both `.woff` variants exist in `public/fonts/` but the `.woff2` variants are missing.

Download the two missing Geist font files in woff2 format from the official Vercel Geist font source. The Geist font is open source (MIT license) and available at:
- https://github.com/vercel/geist-font/raw/main/packages/next/dist/fonts/geist-sans/Geist-Regular.woff2
- https://github.com/vercel/geist-font/raw/main/packages/next/dist/fonts/geist-sans/Geist-Medium.woff2

Use curl or wget to download them directly into `public/fonts/`:

```bash
curl -L "https://github.com/vercel/geist-font/raw/main/packages/next/dist/fonts/geist-sans/Geist-Regular.woff2" -o "public/fonts/Geist-Regular.woff2"
curl -L "https://github.com/vercel/geist-font/raw/main/packages/next/dist/fonts/geist-sans/Geist-Medium.woff2" -o "public/fonts/Geist-Medium.woff2"
```

If the above URLs do not work (404), try the alternative paths:
- https://github.com/vercel/geist-font/raw/main/packages/next/dist/fonts/GeistVF.woff2

As a fallback if network access is unavailable: generate a minimal stub `.woff2` file from the existing `.woff` file using the `woff2` tool if available, OR copy the `.woff` file as a `.woff2` stub so the file exists and the build resolves the reference without warning. The CSS already has a `.woff` fallback in each `@font-face` `src` declaration, so the stub approach will silence the build warning while still serving the correct font via the `.woff` fallback.

**Stub fallback command (if curl fails):**
```bash
cp public/fonts/Geist-Regular.woff public/fonts/Geist-Regular.woff2
cp public/fonts/Geist-Medium.woff public/fonts/Geist-Medium.woff2
```

Note: The stub approach is acceptable only if the real woff2 cannot be downloaded. The `.woff` file will serve correctly in all modern browsers regardless.

Verify the files exist and are non-zero bytes after creation.
