<objective>
Fix `useImageRotation.ts` to safely validate localStorage data with radix-10 parseInt and a NaN guard, and enable `drop_console: true` in `vite.config.ts` to strip all console.log from production bundles.

Purpose: `parseInt(value)` without radix is ambiguous — values starting with `0x` or `0` could be parsed as hex or octal in some engines; a poisoned localStorage value could coerce to NaN and crash `images[NaN]` returning `undefined`. Separately, `console.log` calls left in bundles leak internal state and execution flow information to any user with DevTools open.

Output: `useImageRotation.ts` with safe parseInt(value, 10) + isNaN fallback; `vite.config.ts` with drop_console: true so production build contains zero console.log statements.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → application state | Data read from localStorage is user-controlled and must be treated as untrusted |
| Source code console.log → production bundle | Developer debug output must not reach users in production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | useImageRotation.ts — parseInt without radix | mitigate | Use `parseInt(value, 10)` with explicit radix — prevents ambiguous octal/hex parsing of user-controlled localStorage values |
| T-02-02 | Tampering | useImageRotation.ts — no NaN guard before array index | mitigate | Add `if (!isNaN(parsed))` guard — prevents `images[NaN]` returning `undefined` when localStorage contains a non-numeric string (e.g., injected by XSS on another tab in same origin) |
| T-02-03 | Information Disclosure | vite.config.ts — drop_console: false leaves console.log in production bundle | mitigate | Set `drop_console: true` in terserOptions — removes all console.* calls during the Terser compress pass, preventing internal state/flow information from being visible to users with DevTools |
| T-02-04 | Elevation of Privilege | localStorage key 'designerImageIndex' — out-of-bounds index | accept | Even without the NaN guard, out-of-bounds access returns `undefined` (not a crash). The NaN guard + modulo already cap the index to valid range. No privilege escalation is possible; worst case is the hook shows `undefined` as image src. Risk is informational only. |
</threat_model>
