<task type="auto">
  <name>Task 2: Estabilizar callback de animación en AnimatedElement.tsx con useCallback</name>
  <files>src/components/ui/AnimatedElement.tsx</files>
  <action>
Aplicar PERF-02 a src/components/ui/AnimatedElement.tsx.

El problema: la función `() => { setTimeout(() => { setIsVisible(true); }, delay * 1000); }` se pasa directamente a `useScrollAnimation`. Al ser una función inline, se recrea en cada render de `AnimatedElement`. Si `useScrollAnimation` almacena el callback en una ref o lo usa como dependencia de un useEffect/IntersectionObserver, cada nuevo render destruye y recrea el observer innecesariamente.

La solución: extraer la función a una variable estabilizada con `useCallback`. El único valor externo que captura es `delay`, que viene de las props. Por lo tanto el dep array es `[delay]`.

1. Añadir `useCallback` al import: cambiar `import React, { useState } from 'react'` por `import React, { useState, useCallback } from 'react'`.

2. Antes de la llamada a `useScrollAnimation`, declarar el callback estabilizado:

```tsx
const animationCallback = useCallback(() => {
  setTimeout(() => {
    setIsVisible(true);
  }, delay * 1000);
}, [delay]);
```

3. Pasar `animationCallback` a `useScrollAnimation` en lugar del inline:

```tsx
const elementRef = useScrollAnimation(animationCallback, { threshold, once });
```

NO cambiar nada más del componente — el cast `as React.RefObject<HTMLDivElement>` en el JSX, los estilos, `getAnimationStyles`, y el return permanecen exactamente igual.

Aplicar PERF-03 a src/hooks/useProjectsList.ts.

El problema: el `for...of` con `await` dentro carga cada JSON de proyecto en serie. Con 4 proyectos, el tiempo de carga es la suma de los 4 tiempos individuales. Con `Promise.allSettled` corren en paralelo y el tiempo total es el del más lento.

Reemplazar el bloque `for...of` completo (líneas 26-39) con `Promise.allSettled`. La lógica de manejo de errores por proyecto (el `console.warn`) se preserva usando `result.status === 'rejected'`.

El nuevo bloque dentro de `loadProjects`, reemplazando desde `const projectsData: ProjectCard[] = [];` hasta el cierre del for loop. La línea `const projectIds = [...]` que ya existe en la línea 23 se conserva tal cual — NO volver a declararla:

```ts
const results = await Promise.allSettled(
  projectIds.map(projectId => import(`../data/projects/${projectId}.json`))
);

const projectsData: ProjectCard[] = results
  .map((result, index) => {
    if (result.status === 'rejected') {
      console.warn(`Could not load project ${projectIds[index]}:`, result.reason);
      return null;
    }
    const data: ProjectData = result.value.default;
    return {
      id: data.id,
      cardTitle: data.cardTitle,
      cardTags: data.cardTags,
      heroImage: data.heroImage,
    };
  })
  .filter((p): p is ProjectCard => p !== null);
```

La línea `setProjects(projectsData)` y el bloque `catch` externo y `finally` permanecen sin cambios.

NO cambiar el tipo `ProjectCard`, los imports, ni el `return { projects, loading, error }` al final.
