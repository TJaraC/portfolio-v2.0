Aplicar PERF-06 y PERF-07 a src/components/common/Header.tsx.

**PERF-06 — Throttle de detectHeaderTheme:**

El problema: `detectHeaderTheme` llama a `.getBoundingClientRect()` sobre 6 elementos (header + 6 secciones) en CADA evento de scroll. Con Lenis + scroll nativo, esto puede ser 60-120 llamadas por segundo, cada una forzando layout reflow del browser.

La solución: throttle basado en tiempo con `Date.now()`. Sin dependencias nuevas. 16ms = ~60fps.

1. Añadir una ref para el timestamp del último throttle. Colocar junto a las otras refs, después de `const logoRotation = useRef(0);`:

```tsx
const lastThrottleTime = useRef(0);
```

2. Modificar el `useEffect` del scroll de tema del header (el que tiene `[headerTheme]` como dep array, líneas 107-122). Envolver la llamada a `detectHeaderTheme()` dentro del handler con un guard de throttle:

```tsx
useEffect(() => {
  const handleScroll = () => {
    const now = Date.now();
    if (now - lastThrottleTime.current < 16) return;
    lastThrottleTime.current = now;
    detectHeaderTheme();
  };

  // Detectar tema inicial (sin throttle — solo una vez al montar)
  detectHeaderTheme();

  window.addEventListener('scroll', handleScroll, { passive: true });

  return () => {
    window.removeEventListener('scroll', handleScroll);
  };
}, [headerTheme]);
```

El llamado inicial `detectHeaderTheme()` fuera del handler no tiene throttle — solo el handler en scroll lo tiene.

**PERF-07 — overwrite:true en tween de logo:**

El problema: en el `useEffect` del scroll del logo (líneas 125-149), por cada evento de scroll se llama `gsap.to(logoRef.current, {...})`. Sin `overwrite: true`, GSAP encola un nuevo tween en lugar de reemplazar el anterior. Con scroll rápido, se acumulan docenas de tweens luchando por animar la misma propiedad.

La solución: añadir `overwrite: true` al objeto de opciones del `gsap.to(...)`:

```tsx
gsap.to(logoRef.current, {
  rotation: logoRotation.current,
  duration: 0.3,
  ease: "power2.out",
  overwrite: true,  // <-- añadir esta línea
});
```

`overwrite: true` hace que GSAP mate cualquier tween activo sobre el mismo target/propiedad antes de iniciar el nuevo. Esto es equivalente a llamar `gsap.killTweensOf(logoRef.current, 'rotation')` antes de cada `gsap.to`, pero más eficiente porque GSAP lo hace internamente.

NO cambiar ninguna otra parte de Header.tsx. Los estilos, el markup JSX, los otros useEffects (menú móvil, cleanup general), y la función `scrollToSection` permanecen exactamente igual.
