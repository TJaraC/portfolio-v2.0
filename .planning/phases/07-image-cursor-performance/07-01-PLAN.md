## Objective

Eliminar el tirón al cargar imágenes y la ralentización de hover:
1. CustomCursor: reemplazar querySelectorAll masivo + N event listeners por event delegation en document (2 listeners totales) + requestAnimationFrame para posición
2. ImageWithCurtain: eliminar el useEffect de precarga redundante (src se asignaba 3 veces), poner src en JSX + decoding="async"
3. home.css: añadir will-change + translateZ(0) + backface-visibility a .portfolio-card-img

## Tasks

### Task 1 — CustomCursor: event delegation + rAF

**Problema actual:**
- `querySelectorAll('a, button, [role="button"]...')` selecciona TODO el DOM en cada mount
- Añade mouseenter/mouseleave a cada elemento interactivo individualmente (N listeners)
- `cursor.style.left/top` se muta directamente en cada evento mousemove (sin rAF)

**Cambio:**
- Eliminar querySelectorAll y forEach de addEventListener
- Usar `mouseover`/`mouseout` con `closest()` en document (event delegation — 2 listeners en vez de N)
- Acumular posición en variables y leer desde rAF loop (desacopla mousemove del DOM write)
- Cancelar rAF en cleanup

### Task 2 — ImageWithCurtain: eliminar precarga redundante + decoding="async"

**Problema actual:**
- useEffect crea `new Image()`, asigna src, y TAMBIÉN asigna `imageRef.current.src = src` inmediatamente
- En onload vuelve a asignar `imageRef.current.src = src` (redundante)
- En onerror también (redundante)
- Total: src asignado 3 veces, 2 decode cycles

**Cambio:**
- Eliminar el useEffect de precarga completo
- Añadir `src={src}` y `decoding="async"` directamente al JSX del `<img>`
- La cortina sigue ocultando la imagen hasta que la animación se dispara — el comportamiento visual no cambia

### Task 3 — home.css: GPU hints en portfolio card

**Problema actual:**
- `.portfolio-card-img` no tiene will-change ni GPU layer hints
- El hover `transform: scale(1.05)` no tiene capa preparada

**Cambio:**
- Añadir `will-change: transform`, `transform: translateZ(0)`, `backface-visibility: hidden` a `.portfolio-card-img`
