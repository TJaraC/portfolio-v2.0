```typescript
      }

```

Luego reemplazar el bloque return del useEffect (lineas 76-80 actuales):

```typescript
    return () => {
      if (lenis) {

      }

```

POR:

```typescript
    return () => {
      cancelAnimationFrame(rafId);
      if (lenis) {
        lenis.destroy();
      }
    };
```

NOTA: `rafId` es declarado con `let` dentro del scope de `initializeLenis`.
El cleanup cierra sobre `rafId` via la variable `lenis` local del useEffect.
Pero `initializeLenis` es una funcion interna — `rafId` NO es accesible desde
el return del useEffect directamente.

CORRECCION DE SCOPE: `rafId` debe declararse fuera de `initializeLenis` pero
dentro del `useEffect`, para que el cleanup pueda referenciarlo.

Estructura correcta dentro del useEffect:

```typescript
    let rafId: number;

    const initializeLenis = () => {
      document.documentElement.style.scrollBehavior = 'auto';
      document.body.style.scrollBehavior = 'auto';
      const lenis = new Lenis({
        lerp: 0.1,
        duration: 1.2,
        smoothWheel: true,
        wheelMultiplier: 1,
        touchMultiplier: 2,
        easing: (t: number) => Math.min(1, 1.001 - Math.pow(2, -10 * t))
      });

      setLenisInstance(lenis);
      document.body.classList.add('lenis');

      function raf(time: number) {
        lenis.raf(time);
        rafId = requestAnimationFrame(raf);
      }
      rafId = requestAnimationFrame(raf);

      setTimeout(() => {
        lenis.start();
        lenis.resize();
      }, 100);

      return lenis;
    };

    let lenis: Lenis | null = null;

    handleFontErrors();

    if (document.readyState === 'loading') {
      document.addEventListener('DOMContentLoaded', () => {
        lenis = initializeLenis();
      });
    } else {
      lenis = initializeLenis();
    }

    return () => {
      cancelAnimationFrame(rafId);
      if (lenis) {
        lenis.destroy();
      }
    };
```

La variable `rafId` se declara al nivel del useEffect, antes de `initializeLenis`,
para que el closure del cleanup la pueda leer. La funcion `raf` interna de
`initializeLenis` la escribe via closure. Esto funciona porque `let rafId` esta en
el scope compartido.

CAMBIO EXACTO — eliminar este bloque completo:

```typescript
  // Detectar navegacion de vuelta desde otras paginas y reactivar Lenis
  useEffect(() => {
    const handlePopState = () => {
      // Cuando se usa el boton atras del navegador, reactivar Lenis completamente
      setTimeout(() => {
        if (lenis) {
          // Detener Lenis primero
          lenis.stop();

          // Forzar reinicializacion de event listeners
          setTimeout(() => {
            lenis.start();
            lenis.resize();

            // Forzar reactivacion de smooth wheel
            const wheelEvent = new WheelEvent('wheel', {
              deltaY: 0,
              bubbles: true,
              cancelable: true
            });
            document.dispatchEvent(wheelEvent);
          }, 50);
        }
      }, 100);
    };

    window.addEventListener('popstate', handlePopState);

    return () => {
      window.removeEventListener('popstate', handlePopState);
    };
  }, [lenis]);
```

El resultado es que Home/index.tsx ya no registra ningun listener en `window` para
el evento popstate. App.tsx ya maneja este evento de forma global y completa.

IMPORTANTE: No tocar ningun otro useEffect, import, estado, ni JSX. Solo eliminar
el bloque de useEffect descrito arriba. El resto del archivo queda identico.
