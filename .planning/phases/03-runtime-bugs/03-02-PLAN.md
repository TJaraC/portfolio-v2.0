Cambio de una sola linea en useLenisScroll.ts: el dep array del useEffect.

CAMBIO EXACTO — linea 36 actual:

```typescript
  }, [lenis]);
```

REEMPLAZAR POR:

```typescript
  }, []);
```

El useEffect completo (lineas 19-36) queda asi despues del cambio:

```typescript
  useEffect(() => {
    // Si ya existe la instancia, actualizarla inmediatamente
    if (lenisInstance && !lenis) {
      setLenis(lenisInstance);
    }

    // Registrar callback para cuando Lenis este disponible
    const callback = () => {
      setLenis(lenisInstance);
    };

    lenisCallbacks.push(callback);

    // Cleanup: remover el callback cuando el componente se desmonte
    return () => {
      lenisCallbacks = lenisCallbacks.filter(cb => cb !== callback);
    };
  }, []);
```

RAZON: El effect solo necesita correr una vez al mount. Su objetivo es registrar
un callback en el array global `lenisCallbacks`. La variable `lenis` (estado local)
no es una dependencia real del effect — el callback referencia `lenisInstance`
(variable de modulo, no estado), que no cambia. Con `[lenis]` como dep array,
cada `setLenis(lenisInstance)` causa un re-render, que re-ejecuta el effect, que
llama `setLenis` de nuevo — bucle infinito potencial. Con `[]`, el effect corre
solo al mount, el callback queda registrado, y el cleanup lo elimina al desmontar.

ADVERTENCIA DE ESLINT: Si hay una regla `react-hooks/exhaustive-deps`, el linter
puede quejarse de que `lenis` no esta en el dep array. Esto es un falso positivo
aqui — `lenis` en el body del effect es lectura de estado para evitar una actualizacion
redundante, no una dependencia del comportamiento principal. Si el linter falla
el build, agregar el comentario de supresion en la linea anterior al cierre:

```typescript
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, []);
```

Solo agregar el comentario si `npm run build` falla por este lint. Verificar primero
sin el comentario.
