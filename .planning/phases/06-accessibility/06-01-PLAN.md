## Objective

Make the logo and "back to top" controls fully keyboard-navigable with correct ARIA roles and labels, and update the copyright year to 2026.

## Tasks

### Task 1 — A11Y-01: Make logo keyboard-navigable (Header.tsx)

The logo is currently a `<div>` with `onClick` and `cursor: pointer`. Screen readers and keyboard users cannot interact with it.

**Change:**
- Replace the outer `<div className="logo" ref={logoRef} ...>` with a `<button className="logo" ...>`
- Add `aria-label="Go to home"` on the button
- Remove the inline `style={{ cursor: 'pointer' }}` (buttons already have pointer cursor via CSS or can be styled)
- Update `logoRef` type from `useRef<HTMLDivElement>` to `useRef<HTMLButtonElement>`
- Remove `style={{ cursor: 'pointer' }}` from the element (keep it in CSS if needed)

**Acceptance:** Element is a `<button>` with `aria-label="Go to home"`, reachable via Tab key.

---

### Task 2 — A11Y-02: Make "back to top" keyboard-accessible (Contact.tsx)

The back-to-top element is a `<div>` with only `onClick`. Keyboard users cannot activate it.

**Change:**
- Add `role="button"` to the `<div className="footer-backtop">`
- Add `tabIndex={0}`
- Add `onKeyDown` handler that fires `scrollToTop()` on Enter (key === 'Enter') or Space (key === ' ')

**Acceptance:** Element has `role="button"`, `tabIndex={0}`, and `onKeyDown` that fires on Enter/Space.

---

### Task 3 — A11Y-03: Update copyright year (Contact.tsx)

The footer shows "© 2025 - All Rights Reserved".

**Change:**
- Update to "© 2026 - All Rights Reserved"

**Acceptance:** Search for `2025` in Contact.tsx returns zero results.

## Success Criteria

- [ ] Logo in Header.tsx is a `<button>` with `aria-label="Go to home"`
- [ ] Back-to-top div has `role="button"`, `tabIndex={0}`, `onKeyDown` firing on Enter/Space
- [ ] Copyright shows "© 2026"
- [ ] `tsc --noEmit` passes
